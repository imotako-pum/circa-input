# M3 実装計画：デモサイト

---

## 要件の整理

ROADMAPのM3定義に基づき、以下を実現するデモページを作成する。

| 要件 | 説明 |
|------|------|
| 基本的な使い方（対称モード） | デフォルトの`<circa-input>`でクリック→マージンドラッグの体験 |
| 非対称モードのデモ | `asymmetric`属性付きで両端ハンドルの操作体験 |
| 各種ユースケース | 時間入力・金額入力などの実用的な例 |
| イベント出力のリアルタイム表示 | `input`/`change`イベントの`CircaValue`をライブ表示 |
| 属性を動的に切り替えるコントロールパネル | min/max/step/asymmetric等をUIから変更 |

**完了条件**: `pnpm dev` で起動し、初見のユーザーがページを見て使い方を理解できること。

---

## ページ構成

1ページ構成（シングルページ、セクションスクロール）。理由：
- コンポーネントは1つだけ（`<circa-input>`）
- SPAフレームワークなしのVanilla TS + Viteなので、ルーティングは不要
- セクションごとにアンカーリンクでナビゲーションすれば快適

### セクション構成

```
┌─────────────────────────────────────┐
│  ヘッダー：タイトル + コンセプト説明   │
├─────────────────────────────────────┤
│  Section 1: 基本操作                  │
│   - circa-input（対称モード）         │
│   - 「クリックで値、ドラッグでマージン」 │
│   - リアルタイムCircaValue表示        │
├─────────────────────────────────────┤
│  Section 2: 非対称モード              │

│   - 両端ハンドルの説明               │

│  Section 3: ユースケース集            │

```

## 実装フェーズ

### Phase 1: ページ骨格とスタイリング
**対象ファイル**: `apps/demo/index.html`, `apps/demo/src/style.css`（新規）

- HTMLの全セクション骨格を作成
- ページ全体のCSSスタイリング（シンプルで見やすいデザイン）
- セクション間のナビゲーション（アンカーリンク）
- レスポンシブ対応（モバイルでも見やすく）

**リスク**: LOW

### Phase 2: Section 1 & 2（基本操作 + 非対称モード）
**対象ファイル**: `apps/demo/src/main.ts`

- `@circa-input/web-component`をimportしてカスタム要素を登録
- 各circa-inputに`input`/`change`イベントリスナーを追加
- `CircaValue`をフォーマットしてリアルタイム表示するユーティリティ関数
- 操作ガイド（テキスト + ビジュアル指示）を表示

**リスク**: LOW

### Phase 3: Section 3（ユースケース集）
**対象ファイル**: `apps/demo/src/main.ts`

- 各ユースケースごとに意味のあるラベルとフォーマット
  - 時間 → `14:00 ± 30分` のような表示
  - 金額 → `¥50,000 ± ¥5,000` のような表示
  - 気温 → `25°C ± 3°C` のような表示
- 各ユースケースにCircaValue出力を紐付け

**リスク**: LOW

### Phase 4: Section 4（プレイグラウンド）
**対象ファイル**: `apps/demo/src/main.ts`

- コントロールパネルUI：
  - `min` / `max`（数値入力）
  - `step`（数値入力 or "any"）
  - `asymmetric`（チェックボックス）
  - `margin-max`（数値入力）
  - `disabled`（チェックボックス）
- パネルの値変更時にcirca-inputの属性を動的に更新
- イベントログ：`input`/`change`イベントをタイムスタンプ付きでログ表示（最新N件）

**リスク**: MEDIUM — 属性の動的変更が正しく反映されるか要確認。`attributeChangedCallback`が正しく動作する必要がある。

### Phase 5: Section 5（フォーム統合）
**対象ファイル**: `apps/demo/src/main.ts`

- `<form>` 内に `<circa-input name="demo">` を配置
- submitイベントをインターセプトし、FormDataの中身を表示
- `required`属性のバリデーションデモ

**リスク**: LOW

### Phase 6: CSS変数カスタマイズのデモ（オプション）
**対象ファイル**: `apps/demo/src/main.ts`, `apps/demo/src/style.css`

- CSS Custom Properties（`--circa-*`）のカスタマイズ例
- プレイグラウンドにカラーピッカーやサイズ調整を追加

**リスク**: LOW

## ファイル構成

```
apps/demo/
├── index.html          # 既存 → 全面書き換え
├── src/
│   ├── main.ts         # 既存 → 全面書き換え（エントリポイント）
│   ├── style.css       # 新規（ページ全体のスタイル）
│   ├── sections/       # 新規ディレクトリ
│   │   ├── basic.ts        # Section 1: 基本操作の初期化
│   │   ├── asymmetric.ts   # Section 2: 非対称モードの初期化
│   │   ├── use-cases.ts    # Section 3: ユースケース集
│   │   ├── playground.ts   # Section 4: プレイグラウンド
│   │   └── form.ts         # Section 5: フォーム統合
│   └── utils/          # 新規ディレクトリ
│       └── format.ts       # CircaValue表示フォーマッター
├── package.json        # 既存（変更不要）
└── tsconfig.json       # 既存（変更不要）
```

**設計意図**: `main.ts`に全部書くと肥大化するので、セクションごとにモジュール分割する。各セクションのTSファイルが該当HTMLセクション内のDOM要素にイベントリスナーを設定する構造。

## 依存関係

- 追加パッケージ: **なし**（Vanilla TS + Viteで十分）
- 既存の依存: `@circa-input/core`（型）、`@circa-input/web-component`（カスタム要素登録）

## リスクまとめ

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 属性の動的変更が反映されない可能性 | MEDIUM | Phase 4開始前にWCの`attributeChangedCallback`を確認 |
| モバイルでのレイアウト崩れ | LOW | レスポンシブCSS + 実機確認 |
| WCのスタイルがデモページのCSSと干渉 | LOW | Shadow DOMなので原則隔離されている |
